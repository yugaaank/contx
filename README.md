# contx

let's be real: trying to understand a massive codebase is a certified vibe kill. instead of raw-dogging thousands of lines of code, use `contx` to turn that spaghetti into a structured knowledge vault that plays perfectly with tools like obsidian.

we use tree-sitter AST parsing (big brain energy) to actually understand what your code is doing, not just grep for strings like it's 1995.

## why it's a mood

- **multilingual queen:** python, js/ts, c++, rust, go... we speak 'em all.
- **deep dives only:** we parse the actual AST, so we know the difference between a function call and a random comment.
- **mapping the lore:** we build a graph of imports and calls so you don't get lost in the sauce.
- **visualize with obsidian:** the markdown output is designed to be dropped straight into an obsidian vault. the obsidian graph view will turn your messy repo into a beautiful, interconnected web of logic.
- **not just for obsidian:** while it looks great in an obsidian graph, this is a powerhouse for AI context. instead of making an LLM read every single file (expensive and slow), just point it at the context vault generated by `contx`.
- **agent-ready:** give your AI agents the obsidian-style structured knowledge they need to actually be helpful without hitting token limits.

## getting started (it's easy, i promise)

you can install this globally on any OS.

### 1. prerequisites
- python 3.8+ (don't be that person on 2.7)
- `pipx` installed

### 2. global install
run this in the `context` directory:
```bash
git clone https://github.com/yugaaank/contx.git
cd contx
pipx install .
```

## how to use

once installed, just use the `contx` command. it's that simple.

### create an obsidian-ready vault (markdown)
perfect for when you actually want to read stuff or see that sweet obsidian graph.
```bash
contx /path/to/project #The default folder for output is ~/contx 
contx /path/to/project --out ./my-obsidian-vault #This is for if you need a different output vault
```

### create a JSON index
for when you want to feed your Gemini pipeline or other scripts.
```bash
contx /path/to/project --format json --out ./code-index.json
```

### filter by language
don't let the other files distract you.
```bash
contx /path/to/project --languages python,typescript
```

### search the index
"where did i put that auth logic again?"
```bash
contx --query search "authentication" --index ./code-index.json
```

## the boring bits
by default, we save your context to `~/pipeline-context/<project-name>/`. keep your obsidian files organized.
